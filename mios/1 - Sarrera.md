# Datuen ingeniaritza

## Zer da datuen ingeniaritza?

Datu sistema eta arkitekturen diseinu, sortze eta mantenketan zentratzen den jakintza arloa da.

- Helburua: datu kantitate oso handiak era efiziente eta eskalagarrian jasotzea, gordetzea, prozesatzea eta aztertzea.

## Eginbeharrak

1. Datuen arkitektura diseinatu
    - Datu baseak, data lakeak, data warehouseak
2. Datu hodiak (pipeline-ak) sortu
    - Lan fluxu automatizatuak sortu
    - ETL: Extract, Transform, Load -> mysql, convert, db/datalake/datawarehouse
3. Datuak kudeatu eta optimizatu
    - Kalitatea, integritatea, pribatutasuna eta segurtausna bermatu
    - Almazenamendua eta kontsultak optimizatu
4. Datuen zientzia eta analitika jasan
    - Sarbidea erraztu, erraminta ezberdinekin aztertu ahal izateko

## Garrantzia

- Datuak esponentzialki sortu egiten ari dira, eta DI funtsezkoa da datuetatik balioa ateratzeko
- Erabakiak hartzeko prozesua hobetu, lanak optimizatu, erabiltzaileen esperientzia pertsonalizatu eta negozio aukera berriak lantzen lagundu

## Erraminta eta teknologiak

1. Hodei azpiegiturak
    - AWS, GCP, Azure eta pribatuak
2. Programazio lengoiak
    - Python, java -> pipeline-ak
    - C, C++, Rust -> monitorizaziorako, telemetria datu jasoketa eta mentenketerako
3. Datu base sistemak
    - Erlazionalak (PostgreSQL, mysql)
    - NoSQL
4. Big Data prozesamendu plataformak
    - Hadoop, Spark, denbora errealeko prozesamendu sistemak Apache Kafka
5. ETL eta orkestratze erramintak
    - Apache Airflow
