# Definizioak

## Data Warehouse

Gordailu zentralizatua, iturri askotako datu egituratuak gordetzen dituena informazioa emateko eta aztertzeko.

## Data Mart

Data Warehouse baten azpimultzo txikiago bat, zentratua, departamentu edo negozio batean zentratuta.
![data-mart](data-mart.png)

## Data Lake

Errepositorio zentralizatua, edozein eskalatako datu egituratu eta egituratugabe guztiak gordetzen dituenak.

- Kudeatzeko oso zailak izan daitezke
- Datu hauek kontsultatzeko kluster bat beharrezkoa izan daiteke

## Data Lakehouse

Data Lake baten biltegiratze-malgutasuna eta Data Warehouse baten analisi-gaitasuna konbinatzen duen sistema hibridoa.

- Datuen inguruko taula eta eskemak egitea ahalbidetzen du
- Datu egituratu eta ez egituratuak konbinatzea ahalbidetzen du
- Datuen eguneraketa eta hauek ezabatzea ahalbidetzen du
- Taulen historikoak eta rollback-ak ahalbidetzen ditu
  - Horretarako fitxategien eta metadatuen bertsio zaharrak gordetzen dira

## Data Mesh

Datuen arkitekturarako hurbilketa deszentralizatua, non domeinuek beren datuak produktu gisa dituzten eta erabiltzen dituzten, eskalagarritasuna eta autozerbitzua sustatuz.

## Data Catalog

Erakunde bateko datu guztien metadatuen almazenamendu zentralizatua

- Datuen arteko erlazioak osatzen dute
- Erabiltzaileek datuen deskribapena editatu, datuak, kontsultak eta almazenamendua ikusi dezakete

## Data Model

Datu elementu ezberdinak nola dauden beraien artean erlazionatuak eta datuak
sistema batean nola gorde eta prozesatzen diren

## Data Enrichment

Datu segida edo fluxu bat beste batekin elkartzea

## Data Provenance

Datuen jatorriaren, historiaren eta bizi-ziklo osoko datuen
bilakaeraren erregistro zehatzari dagokio.

- Nola bildu, prozesatu, aztertu eta biltegiratu diren
- Testuinguru kritikoa eskaini (eragina izan dezake interpretazioan, fidagarritasunean...)

## Data Architecture

Datuak erakunde batean nola bildu, biltegiratu, prozesatu eta erabiltzen diren, teknologiak, eroanbideak eta gobernantza barne.
Arkitektua egokiak:

- Segurtasuna
- FIdagarritasuna
- Errendimendu efizentzia
- Kostu optimizazioa
- Automatizaziorako diseinatuta
